public class DataLoader

Summary

The Data Loader class is a very important one and is responsible for all file loading. You can open files without using the data loader by using a filepath from your project's root and piping that path into a resource loader of some type. However, it is VERY EXTREMELY STRONGLY ENCOURAGED to use the data loader AND THE FOLLOWING FILE STRUCTURE for organizing your resources. JE2 uses java's class loader and will pull files relative to that file structure. For example, to access "src/main/resources/Your_Resources!" you would call get("Your_Resources!"); to get the absolute filepath from the resources folder.

HOWEVER I DON'T EVEN RECOMMEND USING THIS METHOD AS TEXTURES AND AUDIO LOADERS JUST REQUIRE A RELATIVE FILEPATH FROM YOUR "src/main/resources" FOLDER!

You really shouldn't have to worry about the DataLoader if you are only using the Texture and AudioSource classes for loading game assets. Just make sure when you pass in a filepath it is relative to "src/main/resources/" because that's what those loaders use! The loaders use this class loader method because it's the easiest way to access files when running from a jar archive.

NOTE: ONLY THE METHODS IN THIS CLASS THAT RETURN BYTE ARRAYS WILL WORK IN A JAR ARCHIVE. FILEPATHS DO NOT WORK BECAUSE OF THE JAR'S ZIP FORMAT!

Constructors

This class has no constructors

Methods

@Nullable public static String getDataFilePath(String path)

path - relative path to resource from "src/main/resources/"
Will convert the relative resource path to an absolute file path.
This will not work in a jar archive.
This will return null if no file is found.

@JarSafe public static byte[] getDataFilePath(String path)

path - relative path to resource from "src/main/resources/"
Will load the file into a byte array which can be used by various resource loaders.
This will work in a jar file.
This will return null if no file is found.
