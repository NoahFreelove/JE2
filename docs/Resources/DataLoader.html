public final class DataLoader

Summary

The Data Loader class is a very important one and is responsible for all file loading. You can open files without using the data loader by using a filepath from your project's root and piping that path into a resource loader of some type. However, it is VERY EXTREMELY STRONGLY ENCOURAGED to use the data loader AND THE FOLLOWING FILE STRUCTURE for organizing your resources. JE2 uses java's class loader and will pull files relative to that file structure. For example, to access "src/main/resources/Your_Resources!" and get the full filepath you would call get("Your_Resources!");. This returns the absolute filepath to your resource in the resources folder.

HOWEVER I DON'T EVEN RECOMMEND USING THIS METHOD AS TEXTURES AND AUDIO LOADERS REQUIRE A RELATIVE FILEPATH FROM YOUR "src/main/resources" FOLDER, NOT ABSOLUTE! SO YOU REALLY DON'T NEED THIS CLASS, IT'S FOR OTHER RESOURCE LOADERS!

Just make sure when you pass in a filepath to any texture or audio loader that it is relative to "src/main/resources/" because that's what those loaders use! The loaders use this class loader method because it's the easiest way to access files when running from a jar archive.

Here's an example of how I would load the image "src/main/resources/images/image.png" into a Texture object using checkExistElseCreate(String resourceName, int resourceID, String filePath);

Texture texture = Texture.checkExistElseCreate("texture", -1, "images/image.png");

That's It!

NOTE: ONLY THE METHODS IN THIS CLASS THAT RETURN BYTE ARRAYS WILL WORK IN A JAR ARCHIVE. FILEPATHS DO NOT WORK BECAUSE OF THE JAR'S ZIP FORMAT!

Constructors

This class has no constructors

Methods

public static String getDataFilePath(String path)

path - relative path to resource from "src/main/resources/"
Will use the relative resource path given and use the ResourceLoader to get it's absolute filepath in your system.
This will not work in a jar archive.
This will return empty string if no file is found.

@Nullable @JarSafe public static String getDataFile(String path)

path - relative path to resource from "src/main/resources/"
Will use the relative resource path given and use the ResourceLoader to return a file object.
This will work in a jar archive.
This will return null if no file is found.

@JarSafe public static byte[] getDataBytes(String path)

path - relative path to resource from "src/main/resources/"
Will use the ResourceLoader and the relative filepath given to process a file's data into a byte array which can be used by various resource loaders.
This will work in a jar archive.
This will return empty byte array if no file is found.

@JarSafe public static String[] getBytesAsString(String path)

Returns getBytes[] in a String format

@Nullable public static String get(String path)

Gets the absolute filepath of a resource with the "/" prefix. Should be the default get method you use for filepaths

@JarSafe public static byte[] getBytes(String path)

Gets the data of a file with the "/" prefix. Should be the default get method you use for getting file data

public static String[] readTextFile(String path)

Reads a file's content into a string array split by newlines.
