public enum ResourceLoadingPolicy

Summary

The ResourceLoadingPolicy is the policy which the ResourceManager uses when indexing your resources. The policy dicates how the ResourceManager will compare and check other resources when attempting to load a new resource. It will see if identical (or assumed identical) resources already exist in memory and return a reference to those instead of using memory to create duplicate data.

Constructors

This class has no constructors.

Methods

This class has no methods.

Fields

DONT_CHECK_IF_EXISTS

ResourceManager will not do any checks for duplicate resources when a new resource is created. Every resource will assume it's unique. This is the least memory conscious policy.

CHECK_BY_NAME

ResourceManager will see if another resource by the same name exists. Will go through all indexed resources in a linear search to see if the created resource already exists. You are trading a little bit of procssing time to save you creating duplicate memory.

CHECK_BY_ID

ResourceManager will see if another resource by the same ID exists.

CHECK_BY_BYTE_DATA

By far the slowest option but useful if you really want to save memory. Will check each resource and compare byte for byte if they are identical. If a match is found it will not allocate new space for the resource but return a reference to the already created one.
