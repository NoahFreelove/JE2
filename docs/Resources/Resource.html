Resources

public class Resource<T extends ResourceBundle > implements Serializable

Summary

The Resource is the primary way to store and access file data. A resource has three primary components, Its Bundle (what actually holds the byte data), its Name, and its ID. The last two are for comparing against other Resources to see if identical ones already exist in memory.

You probably wont interact with Resources directly as you will likely use the Texture object for images and AudioSource for sound, both handle this for you.

Constructors

private Resource()

Don't let anyone create a resource without giving it a Name and ID.

public Resource (T bundle, String name, int ID)

returns a new instance of Resource and sets fields.
A resource's ID can always be changes but it's name is final.

Methods

@Nullable public T getBundle()

Returns the Resource's bundle, the object that contains the actual byte data.

@Override public boolean equals(Object obj)

Will compare two resources to check if they are identical or if one already exists.
Depending on the ResourceManager policy, it will compare by Name, ID, Or check byte for byte. This is to conserve memory.

It is assumed that if two Resources have the same name OR ID that they are identical. Byte for byte resource checks aren't the default and are slow.
