Resources

public class ResourceManager implements Serializable

Summary

The ResourceManager manages and indexes everything to do with resources and data loading. Whenever you create a new Texture or AudioSource it along with it's name and ID get indexed in the ResourceManager. This ensures that there isn't memory being taken up by lost resource references and that you aren't loading duplicate data when not required.
The ResourceManager also supports warming up assets which means you can pass it a large array of Resources and have them all loaded and ready to be used in game.

Constructors

This class has no constructors.

Methods

public static void indexResource(Resource<?> resource)

Registers and indexes the resource to the appropriate registry of textures or sounds.

public static Resource<?> getByName(String name , Class<?> type)

Will search textures or sounds (depending on the type class you give it) for a resource with the same name.

As of right now, type can only be TextureBundle.class or AudioBundle.class

public static Resource<?> getByID(String name , Class<?> type)

Will search textures or sounds (depending on the type class you give it) for a resource with the same ID.

As of right now, type can only be TextureBundle.class or AudioBundle.class

@Nullable public static Resource<?> getIfExists(Resource<?> resource)

Will check if an identical OBJECT exists in the textures or sounds registry.
Once again this method checks by object comparison and will not take name or ID into account regardless of your policy.
This method will return null if the policy is set to DONT_CHECK_IF_EXISTS (for obvioud reasons) and null if no match is found.

@Nullable public static Resource<?> getIfExists(Class<? extends ResourceBundle> clazz, String name, int ID)

Based on the type of resource this method will search through the indexed textures or sounds and check if their ID or names match (depends on the policy set)
If a match is found this method will return that resource's reference, otherwise it'll return null.

@JarSafe public static void warmupAssets(String[] names, String[] assets, Class<? extends ResourceBundle>[] classes)

names - the names for the resources, to ensure them being unique you can just set them to the same as their filepath.
assets - the relative filepaths to the assets.
classes - the resource classes.

@JarSafe public static void warmupAssets(Triplet<String, String, Class<? extends ResourceBundle>>[] assets)

assets - Array of assets to warmup. First parameter is the name, second is filepath, third is class

@JarSafe public static void warmupAsset(String name, String asset, Class<? extends ResourceBundle> clazz)

name - the name for the resource, to ensure its unique you can just set it to the same as it's filepath.
asset - the relative filepath to the asset.
clazz - the resource's class.

public static void warmupQueue()

Warms up the entire warmupQueue by creating them as textures or audio sources and registering them for use next time a new resource with the same name attempts to create itself.
